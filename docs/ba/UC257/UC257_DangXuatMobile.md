## UC257 — Đăng xuất ứng dụng Mobile

### 1. Mô tả chức năng

**Tên chức năng:** Đăng xuất ứng dụng Mobile
**Mô tả:** Chức năng cho phép người dùng đã đăng nhập kết thúc phiên làm việc hiện tại, xóa thông tin phiên đăng nhập khỏi thiết bị và quay trở về màn hình Đăng nhập.
**Phân quyền:** Toàn bộ người dùng đang đăng nhập.
**Truy cập chức năng:** Menu chính (Sidebar) → Mục "Cấu hình tài khoản" → Màn hình Cài đặt → Nút "Đăng xuất".
**Điều kiện tiên quyết (Preconditions):** Người dùng đang ở trạng thái đã đăng nhập và phiên làm việc còn hiệu lực.
**Điều kiện kết thúc (Postconditions):** Thông tin phiên làm việc, token bảo mật, dữ liệu cache và Push Token bị xóa khỏi thiết bị; hệ thống chuyển hướng về màn hình Đăng nhập.
**Chức năng đáp ứng usecase số:** UC257 (Phụ lục XIV — STT 257)

### 3. Mô tả các xử lý của chức năng

#### 3.1 Xử lý Đăng xuất

1. **Xác nhận:** Hiển thị Dialog xác nhận.
   - **Outside Tap:** Tap vào vùng overlay bên ngoài Dialog **KHÔNG đóng dialog**. Đây là thiết kế có chủ đích (deliberate design) — thao tác đăng xuất là hành động nguy hiểm, cần người dùng xác nhận rõ ràng bằng nút bấm, tránh vô tình đóng dialog và lỡ tay bấm nhầm "Đăng xuất".
   - **Android Back button:** Đóng dialog (không đăng xuất).
2. **Hủy đăng ký Push Token (SDK):** Song song với việc gọi API logout, Client gọi SDK tương ứng (`FCM deleteToken()` / `APNS unregisterForRemoteNotifications`) để ngừng nhận push ngay lập tức.
   - **Timing:** SDK unregister call và API logout call là **độc lập với nhau, không block nhau**.
   - **Không ảnh hưởng nếu SDK call fail:** Nếu SDK call thất bại (VD: FCM không response), app vẫn tiếp tục xử lý logout bình thường. Token phía server sẽ được revoke qua API logout.
3. **Gọi API Logout:** Hệ thống hiển thị **Button Loading** (spinner inline trên nút "Xác nhận", nút chuyển sang trạng thái disabled) và gọi API thu hồi token trên Server.
4. **Dọn dẹp (Client):** Sau khi API phản hồi thành công (hoặc gặp lỗi 401/timeout/mạng), xóa sạch Token, User Info, Secure Storage, Cache trên thiết bị.
   - **Note (Idempotent):** Thao tác logout là idempotent operation. Việc logout trên thiết bị này sẽ revoke token trên **tất cả các thiết bị khác** của cùng user account. Các thiết bị còn lại sẽ bị đẩy về màn hình Đăng nhập khi token bị revoke.
5. **Điều hướng:** Về màn hình Đăng nhập. Hiển thị Toast thông báo: *"Đăng xuất thành công."* (3 giây — áp dụng thống nhất cho cả luồng API thành công lẫn fallback). **(Postcondition)**

**Lưu ý về Outside Tap:** Các Bottom Sheet / Modal Filter (theo CMR-02 và UC69, UC70, UC56-57-66-68) cho phép tap ngoài để đóng — nhưng **Dialog xác nhận đăng xuất** là ngoại lệ, KHÔNG cho phép đóng bằng tap ngoài. Lý do: hành vi đăng xuất là thao tác nghiêm trọng (cần xác nhận chủ động), không phải thao tác hủy bỏ lọc/filter.

#### 3.2 Xử lý Android Back button (Hardware)

- **Tại màn hình Cài đặt:** Tap Back button → Quay về màn hình Trang chủ (Home).
- **Khi Dialog xác nhận đang mở:** Tap Back button → Đóng Dialog (giữ nguyên tại màn hình Cài đặt).
- **Sau khi đăng xuất hoàn tất (luồng 3.1):** Đã về màn hình Đăng nhập → Back button không có hành động gì (không quay về màn trước đó vì đã logout).
- **Sau khi Đổi mật khẩu thành công (auto-logout từ UC253):** Đã về màn hình Đăng nhập → Back button không có hành động gì (không quay lại UC253 hay màn hình Cài đặt).

**Lưu ý về iOS Gesture Navigation:** Ứng dụng không xử lý iOS swipe-from-left-edge gesture để điều hướng back — iOS tự xử lý ở tầng OS. Khi Dialog xác nhận đang mở, swipe gesture sẽ không đóng dialog (tương tự Outside Tap rule).

#### 3.4 Xử lý Ngoại lệ

- **Timeout (API Logout):** Quá **10 giây** (CMR-16) → Thực hiện Fallback: Xóa local data, điều hướng Login, hiển thị Toast: *"Đăng xuất thành công."* (3 giây — thống nhất với luồng thành công).
- **Lỗi 401 (Token hết hạn khi đang Logout):** Vẫn gọi API -> Nhận lỗi 401 -> Áp dụng CMR-07: Hiển thị toast *"Phiên đăng nhập hết hạn"*, xóa local và điều hướng về Login.
- **Lỗi mạng:** Thực hiện quy trình Fallback tương tự trường hợp Timeout.

### 4. Tiêu chí chấp nhận (Acceptance Criteria)

- **AC1:** Dialog xác nhận phải được phản hồi bằng nút bấm (không được đóng bằng tap ngoài).
- **AC2:** Phải hiển thị Button Loading (spinner trên nút "Xác nhận", nút disabled) trong khi đang xử lý API logout.
- **AC3:** App phải **gọi** SDK unregister call khi logout; luồng logout không bị block dù SDK call fail.
- **AC4:** Hệ thống phải điều hướng về màn hình Login ngay cả khi API logout gặp lỗi mạng hoặc timeout 10s.
- **AC5:** Thông tin người dùng (Tên, Email) phải được xử lý hiển thị đúng trong các trường hợp dữ liệu quá dài (truncate). Trường Tên không cho phép chỉnh sửa nên không có trường hợp null.
- **AC6:** Sau khi đổi mật khẩu thành công tại UC253, hệ thống phải: (1) hiển thị Toast *"Đổi mật khẩu thành công. Vui lòng đăng nhập lại."* (3 giây), (2) xóa local data (Token, User Info, Secure Storage, Cache), và (3) điều hướng về màn hình Đăng nhập. Nếu người dùng bấm Back từ UC253 mà chưa đổi MK → quay về màn hình Cài đặt (phiên đăng nhập vẫn giữ nguyên). Android Back button tại màn hình Đăng nhập sau khi đổi MK thành công không có hành động gì.
